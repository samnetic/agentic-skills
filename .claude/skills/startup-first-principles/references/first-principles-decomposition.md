# First-Principles Decomposition

## Goal

Reduce strategic ambiguity by decomposing a startup decision into invariant mechanics.

## Decomposition Steps

1. Define the unit of value.
2. Define the primary actor receiving value.
3. Define what must be true for repeatable delivery.
4. Define what must be true for repeatable capture of value.
5. Define what must be true for repeatable distribution.

## Worksheet

| Dimension | Prompt | Example |
|---|---|---|
| Value creation | What concrete outcome do users buy? | Reduce reporting time by 70% |
| User segment | Who has urgent pain and budget? | RevOps managers at B2B SaaS (50-500 employees) |
| Mechanism | Why does the product work? | Automated data stitching with deterministic mapping |
| Distribution | How do first 100 users reliably arrive? | Partner channel + founder-led outbound |
| Capture | Why do users pay and keep paying? | Recurring workflow dependency + measurable ROI |

## Constraint Mapping

Classify constraints before ideation:

- Hard constraints: cash runway, team size, compliance rules.
- Soft constraints: channel preference, feature preferences, internal habits.

Only optimize inside hard constraints. Challenge soft constraints.

## Bottleneck Identification

Pick one dominant bottleneck:

- Demand bottleneck
- Activation bottleneck
- Retention bottleneck
- Monetization bottleneck
- Delivery bottleneck

Run strategy work against that bottleneck first.

## Output

Produce a one-page decomposition with:

- Top 3 assumptions
- Top bottleneck
- One decision to make this cycle
