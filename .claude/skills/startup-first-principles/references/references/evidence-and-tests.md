# Evidence and Validation Tests

## Evidence Tiers

| Tier | Type | Trust Level |
|---|---|---|
| E0 | Opinion / intuition | Very low |
| E1 | Anecdotal user quotes | Low |
| E2 | Structured interviews with consistent pattern | Medium |
| E3 | Behavioral evidence (usage or willingness to pay) | High |
| E4 | Repeated behavior across cohorts and time | Very high |

Do not make irreversible decisions using only E0-E1 evidence.

## Hypothesis Format

Use this statement:

`If [segment] experiences [pain], then [offer] will produce [measurable result] within [time], because [mechanism].`

Example:

`If RevOps managers waste time reconciling CRM and billing data, then a daily auto-reconciled dashboard will reduce weekly reporting time by 60% within 14 days, because manual mapping is eliminated.`

## Validation Test Catalog

| Test | Cost | Speed | Best Use |
|---|---|---|---|
| Problem interview | Low | Fast | Validate pain severity |
| Concierge pilot | Medium | Fast | Validate value before automation |
| Wizard-of-oz flow | Medium | Fast | Validate demand for automation |
| Paid smoke test | Low | Fast | Validate message-market resonance |
| Pre-sell offer | Medium | Medium | Validate willingness to pay |
| Narrow beta | Medium | Medium | Validate retention mechanics |

## Decision Rules

Define these before each test:

- Primary success threshold
- Hard failure threshold
- Max spend/time limit
- Follow-up decision for each outcome

## Stop Conditions

Stop an initiative when one or more apply:

- Two consecutive tests fail the hard threshold.
- New data invalidates core mechanism assumptions.
- Opportunity cost exceeds expected upside versus alternatives.
- No plausible path to distribution emerges within current runway.
